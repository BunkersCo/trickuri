Welcome to Trickuri!

Trickuri has two types of test cases: tricky URLs and web platform tests.

Tricky URLs

Load these URLs in a web browser to test how the browser's main URL display handles them.

  • Baseline case (example.test) (https)
  • Baseline case (example.test) (http)
  • RTL url with LTR tld (https)
  • RTL url with RTL tld (http)
  • LTR url with RTL tld (https)
  • RTL url with LTR tld (http)
  • RTL url with RTL tld (http)
  • LTR url with RTL tld (http)
  • Extremely long URL to trigger elision, ending in an url that looks similar to example.test
  • Extremely long URL to trigger elision, starting with a url that looks similar to example.test
  • URL uses cyrillic е instead of e in example.test
  • URL uses number one instead of l in example.test
  • Evil.test, for testing paths do not cause the evil.test origin to be hidden
  • A very long path ending in example.test

Web platform tests

These tests exercise individual web platform features to test how they display URLs. After loading one of the tricky URLs above, exercise these web platform tests to observe how the individual features display the tricky URL.
